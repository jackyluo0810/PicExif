# 图片地理位置信息提取应用 - 需求文档和开发计划

## 项目概述

基于Windows 10的桌面应用，用于提取图片中的EXIF数据，特别是地理位置信息，并提供友好的用户界面。

## 功能需求

### 核心功能
1. **图片上传功能**
   - 支持拖拽上传图片
   - 支持点击按钮选择图片上传
   - 支持常见图片格式（JPEG、PNG、BMP等）

2. **图片预览功能**
   - 上传后显示图片缩略图
   - 支持缩略图缩放和查看

3. **EXIF数据提取功能**
   - 提取图片名称
   - 提取创建时间
   - 提取分辨率信息
   - 提取照相机型号
   - 提取地理位置信息（经纬度）

4. **信息显示和复制功能**
   - 侧边栏显示提取的图片信息
   - 每个信息项旁边提供复制按钮
   - 点击复制按钮可将对应信息复制到剪贴板

### 用户界面需求
- 主界面分为两个主要区域：图片上传/预览区和信息显示区
- 图片上传区支持拖拽和点击上传
- 信息显示区采用清晰的列表布局
- 整体界面简洁美观，操作直观

## 技术选型

### 开发框架选择
基于现有项目结构分析，建议使用以下技术栈：

**方案一：WPF (.NET 6.0)**
- 优势：
  - 与现有项目技术栈一致（已有DeskToy WPF项目）
  - 原生Windows桌面应用，性能优秀
  - 支持拖拽操作和丰富的UI控件
  - 易于打包为可执行文件

**方案二：Electron + React**
- 优势：
  - 跨平台兼容性更好
  - 现代Web技术栈，开发效率高
  - 与现有Next.js项目技术栈相似

**推荐方案：WPF (.NET 6.0)**
- 更适合Windows桌面应用
- 与现有项目技术栈一致
- 更好的系统集成能力

### 关键技术组件
1. **EXIF数据提取**：使用`System.Drawing`或第三方库如`MetadataExtractor`
2. **图片处理**：使用`System.Drawing`进行缩略图生成
3. **UI框架**：WPF XAML
4. **剪贴板操作**：使用`System.Windows.Clipboard`

## 风险评估和应对措施

### 技术风险
1. **EXIF数据提取兼容性**：不同图片格式的EXIF数据可能存在差异
   - 应对：使用成熟的第三方库，添加详细的错误处理

2. **地理位置信息格式**：不同相机的地理位置信息格式可能不同
   - 应对：实现多种格式的解析，提供标准化的输出

3. **性能问题**：大图片处理可能影响性能
   - 应对：添加图片大小限制，使用异步处理

### 用户体验风险
1. **操作复杂性**：功能过多可能导致界面复杂
   - 应对：保持界面简洁，提供清晰的操作指引

## 验收标准

1. 应用能够正确启动并显示主界面
2. 支持拖拽和点击两种上传方式
3. 能够正确显示图片缩略图
4. 能够提取并显示所有要求的EXIF信息
5. 复制功能正常工作
6. 应用在Windows 10上稳定运行
